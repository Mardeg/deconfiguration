Compact Flash card erasure setup

Until now we have been unable to use Blancco to erase (first gen) CF cards as they are in networking devices with no video output, so those cards have been destroyed. The problem this brings is that CF cards are no longer being manufactured so when we sell those devices without their CF cards our customers have trouble replacing them with whatever they can buy which is increasingly expensive as the remaining stockpile sellers have reduces and will eventually run out.

This page documents a setup to successfully erase CF cards using the existing Blancco RPD USB bootdrives we use for other storage media, providing the same erasure reports for Makor.

NOTE: The "Random Byte Overwrite (3x)" erasure method is the one which matches the ASD requirements for up to and including PROTECTED level, so that is what the steps below will set.

Adjustments to the ZimaBlade BIOS

For the ZimaBlade to recognise CF cards, you initially must follow these steps:

  1. To access the BIOS continuously press the DEL key while powering on the ZimaBlade.
  2. Navigate to the Chipset tab using the arrow keys.
  3. Select South Cluster Configuration → PCI Express Configuration, ensure these options are set:
    • PCI Express Clock Gating: Disabled
    • Peer Memory Write Enable: Enabled
    • Open the PCI Express Root Port 3 submenu and apply these settings:
      • ASPM (Active State Power Management): Disabled
      • PCIe Speed: Force to Gen 1
      • Transmit Half Swing: Enabled
  4. Navigate to the Boot tab, CSM Support: Enabled, also set Storage and Other PCI devices to: Enable
  5. Navigate to the Save & Exit tab, select Save Changes and Reset.

Booting and operating Blancco

  1. Plug the CF card from a device into the adapter you've connected to the ZimaBlade, ensuring the LED on it lights up when you turn the power on.
  2. Plug in the Blancco RPD (Process: Manual) USB bootdrive into a free port (in the keyboard hub works for me) then continually press F11 while powering up to bring up the boot menu.
  3. Select the non-UEFI USB option then Remove Freeze Lock to boot into Blancco.
  4. When the Devices interface comes up ensure the CF card is selected, and DESELECT the built-in Samsung (31.27 GB) drive that the ZimaBlade comes with.
    (Open this link in a new browser tab if the embed fails)
  5. Click ⚙️ (CTRL+G) at the bottom to open the Erasure Settings dialog and navigate to the "In-process" tab.
  6. From the "Erasure standard" dropdown select "Random Byte Overwrite (3x)"
  7. In that same dialog drag the "Verification level" slider up to 100%. This read back pass is vital for ASD compliance.
  8. Then click Apply (CTRL+ENTER) to close the dialog.
  9. Go to the Custom Fields tab (CTRL+2) and manually enter the order number and asset tag ID of the device from which you're erasing the CF card, and your user number.
