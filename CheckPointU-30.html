Check Point U-30

  1. Start the console session.

    Plug in serial cable to console port of device from PC COM1 port
    Open PuTTY and select "Serial" connection type, leave baud rate (Speed) and other settings as default.
    Plug power into device and if a power button exists, turn it on.

  2. Erase the current configuration.

    Using the arrow keys select the line Reset to factory defaults with the highest version number, press Enter and confirm with y:
    Check Point SecurePlatform R75.40 +-------------------------------------------------------------------------+ | Start in normal mode | | Start in maintenance mode | | Start in online debug mode | | Start in offline debug mode | | HW Diagnostics | | Revert from other image | | Start in 64bit mode | | Start in 64bit online debug mode | | Start in 64bit offline debug mode | | Reset to factory defaults - R75.40 | | Reset to factory defaults - R71.40 | | | +-------------------------------------------------------------------------+ Use the ^ and v keys to select an entry. Press enter to choose the boot option. This will erase the entire configuration. Do you wish to continue [no]: y Selected: Reset to factory defaults - R75.40
    Further output will take you to the login:  prompt.

  3. Check that default credentials work.

    Type admin for both the login name and the password:
    Check Point SecurePlatform R75.40 For Web User Interface access connect to https://192.168.1.1:4434 login: admin Password: admin You must change the default password.
    Do not continue, just turn off/unplug power and console cables from the device.

  4. Copy all the output from the session into your records.

    Right-click PuTTY's titlebar, from that dropdown menu click on "Copy All to Clipboard"
    Open the file of record certifying the deconfiguration of this device and use CTRL V to paste all the output from PuTTY into that file. Formatting the text with a fixed-width font helps it look more readable.

  5. End the session.

    Close PuTTy.
