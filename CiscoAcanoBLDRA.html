Cisco Acano BLDRA

  1. Start the console session.

    Plug in serial cable to device from PC COM1 port
    Open PuTTY and select "Serial" connection type, leave baud rate (Speed) and other settings as default.
    Plug power into device and if a power button exists, turn it on.

  2. Reset admin password.

    Allow the system to boot up to the login:  prompt and check the default admin/admin for login/password fails:
    Welcome to the CMS MMP mmp.testname login: admin Please enter password: admin Login incorrect
    Unscrew the front top hatch and press the red RESET button:
    Type admin for the login, you should not be prompted for a password if you did this within 4 minutes of pressing the red reset button:
    Welcome to the CMS MMP mmp.testname login: admin mmp.testname>

  3. Erase the configuration.

    Type the factory_reset full command and confirm with capital Y (not holding the Shift button down when typing Y results in nothing happening):
    mmp.testname> factory_reset full WARNING!!! This command will return your system to factory state This means that you will lose (a) all configuration data (b) remote access to the system After completion the system will reboot Consider using "backup" before proceeding Are you sure you wish to proceed? (Y/n) Y mmp.testname> The system is going down NOW! Sent SIGTERM to all processes Sent SIGKILL to all processes Requesting system reboot Restarting system. U-Boot 2012.04.01 (Acano 31-0001-02-U)

  4. Copy all the output from the session into your records.

    Right-click PuTTY's titlebar, from that dropdown menu click on "Copy All to Clipboard"
    Open the file of record certifying the deconfiguration of this device and use CTRL V to paste all the output from PuTTY into that file. Formatting the text with a fixed-width font helps it look more readable.

  5. End the session.

    Turn off/unplug the device and close PuTTy.
