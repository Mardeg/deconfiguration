Cisco N5K-C5548P NEXUS 5500

  1. Start the console session.

    Plug in serial cable to device from PC COM1 port
    Open PuTTY and select "Serial" connection type, leave baud rate (Speed) and other settings as default.
    Plug power into device and if a power button exists, turn it on.

  2. List the directory from the loader>  prompt.

    Repeatedly press the keys Ctrl L to break the boot sequence, then type the dir command (shown below as yellow text). Example output:
    Version 2.00.1201. Copyright (C) 2009 American Megatrends, Inc. User break with default console settings loader> dir bootflash: n5000-uk9-kickstart.5.1.3.N2.1.bin n5000-uk9.5.1.3.N2.1.bin mts.log vdc_2 vdc_3 vdc_4 20090101_152843_poap_3438_init.log license_SSI164405BA_6.lic 20130211_154154_poap_3365_init.log 20130809_012222_poap_3365_init.log vlan.dat n5000-uk9-kickstart.5.1.3.N2.1b.bin n5000-uk9.5.1.3.N2.1b.bin
    Take note of the filename ending in .bin that has kickstart in it for the next step.
    Also note the corresponding .bin file with a name that matches the above without kickstart in its filename. This is the OS image file to be used in step 4.

  3. Manually boot from the kickstart image.

    Use the filename from the previous step with the boot command:
    loader> boot bootflash:n5000-uk9-kickstart.5.1.3.N2.1b.bin Booting kickstart image: bootflash:n5000-uk9-kickstart.5.1.3.N2.1b.bin.... ............................................................................... .................................Image verification OK
    Further output should bring you to the switch(boot)#  prompt.

  4. Set a temporary admin password and load the OS image

    Type the config terminal command to enter configuration mode:
    switch(boot)# config terminal Enter configuration commands, one per line. End with CNTL/Z. switch(boot)(config)#
    Type the admin-password command using a password that complies with the expected minimum standard, so include at least one uppercase letter and a number avoiding dictionary words, then exit configuration mode. Example output:
    switch(boot)(config)# admin-password Passw0rd123 WARNING! Enabling local authentication for login through console due to password recovery switch(boot)(config)# exit
    Use the noted OS image from the directory listing with the load command:
    switch(boot)# load n5000-uk9.5.1.3.N2.1b.bin Uncompressing system image: bootflash:/n5000-uk9.5.1.3.N2.1b.bin Thu Jan 1 11:05:41 AEST 2009 Loading plugin 0: core_plugin... Loading plugin 1: eth_plugin... ethernet switching mode INIT: Switching to runlevel: 3
    Further output should bring you to the login:  prompt.

  5. Gain admin access to the commandline.

    Log in with username admin and the password you set in the previous step:
    bwy-omisp-r11 login: admin Password: Passw0rd123 Cisco Nexus Operating System (NX-OS) Software TAC support: http://www.cisco.com/tac
    Further output should bring you to the commandline prompt with admin privileges.

  6. Erase log files, vlan data and all configuration settings.

    Type the dir command, then the del command with * where necessary to match multiple filenames listed:
    bwy-omisp-r11# dir 0 Jan 02 02:28:43 2009 20090101_152843_poap_3438_init.log 0 Feb 12 02:41:54 2013 20130211_154154_poap_3365_init.log 0 Aug 09 11:22:22 2013 20130809_012222_poap_3365_init.log 516 Feb 12 02:31:18 2013 license_SSI164405BA_6.lic 4096 Jan 01 11:03:27 2009 lost+found/ 3131 Jan 01 11:06:03 2009 mts.log 34357760 Feb 12 02:24:01 2013 n5000-uk9-kickstart.5.1.3.N2.1.bin 34349568 Aug 15 09:26:29 2013 n5000-uk9-kickstart.5.1.3.N2.1b.bin 172662417 Feb 12 02:24:49 2013 n5000-uk9.5.1.3.N2.1.bin 172705188 Aug 15 09:32:08 2013 n5000-uk9.5.1.3.N2.1b.bin 4096 Jan 02 02:27:46 2009 vdc_2/ 4096 Jan 02 02:27:46 2009 vdc_3/ 4096 Jan 02 02:27:46 2009 vdc_4/ 748 Jan 01 11:06:26 2009 vlan.dat Usage for bootflash:// 532013056 bytes used 1118892032 bytes free 1650905088 bytes total bwy-omisp-r11# del 20* This command may delete one or more files. Do you want to continue? (yes/no) [y] y Do you want to delete "20090101_152843_poap_3438_init.log" ? (yes/no/abort) [y] y Do you want to delete "20130211_154154_poap_3365_init.log" ? (yes/no/abort) [y] y Do you want to delete "20130809_012222_poap_3365_init.log" ? (yes/no/abort) [y] y bwy-omisp-r11# del mts.log bwy-omisp-r11# rmdir lost+found bwy-omisp-r11# del vlan.dat
    Note the name of files ending in .lic if any for the next step. Keep these files, do not delete them.
    Type the write erase command and confirm with y.
    bwy-omisp-r11# write erase Warning: This command will erase the startup-configuration. Do you wish to proceed anyway? (y/n) [n] y
    The temporary password you set earlier is now erased.

  7. Optionally list any license keys and the services they enable.

    Type the show license file command followed by the .lic filename noted in the previous step:
    bwy-omisp-r11# show license file license_SSI164405BA_6.lic license_SSI164405BA_6.lic: SERVER this_host ANY VENDOR cisco INCREMENT LAN_BASE_SERVICES_PKG cisco 1.0 permanent uncounted \ VENDOR_STRING=MDS HOSTID=VDH=SSI164405BA \ NOTICE=<LicFileID>20130211073027000</LicFileID><LicLineID>1</LicLineID> <PAK>N5K-C5548UP-FASSI164405BA</PAK> \ SIGN=3E8168AEBA50 INCREMENT LAN_ENTERPRISE_SERVICES_PKG cisco 1.0 permanent uncounted \ VENDOR_STRING=MDS HOSTID=VDH=SSI164405BA \ NOTICE=<LicFileID>20130211073027000</LicFileID><LicLineID>2</LicLineID> <PAK>N5K-C5548UP-FASSI164405BA</PAK> \ SIGN=ADEE1600C15A
    Type the show license usage command:
    bwy-omisp-r11# show license usage Feature Ins Lic Status Expiry Date Comments Count -------------------------------------------------------------------------------- FCOE_NPV_PKG No - Unused - FM_SERVER_PKG No - Unused - ENTERPRISE_PKG No - Unused - FC_FEATURES_PKG No - Unused - VMFEX_FEATURE_PKG No - Unused - ENHANCED_LAYER2_PKG No - Unused - LAN_BASE_SERVICES_PKG Yes - In use Never - LAN_ENTERPRISE_SERVICES_PKG Yes - Unused Never - --------------------------------------------------------------------------------

  8. Reboot the device and check that it boots to first-run setup prompts.

    Type the reload command and confirm with y:
    bwy-omisp-r11# reload WARNING: This command will reboot the system Do you want to continue? (y/n) [n] y Shutdown Ports.. writing reset reason 9, Broadcast message from root (ttyS0) (Thu Jan 1 11:08:49 2009): INIT: Sending processes the TERM signal Sending all processes the TERM signal... Sending all processes the KILL signal... Unmounting filesystems... Resetting board
    Wait for it to finish booting and shows the following prompts:
    Abort Power On Auto Provisioning and continue with normal setup ?(yes/no)[n]: y ---- System Admin Account Setup ---- Do you want to enforce secure password standard (yes/no):
    Do not continue, just turn off/unplug the device.

  9. Copy all the output from the session into your records.

    Right-click PuTTY's titlebar, from that dropdown menu click on "Copy All to Clipboard"
    Open the file of record certifying the deconfiguration of this device and use CTRL V to paste all the output from PuTTY into that file. Formatting the text with a fixed-width font helps it look more readable.

  10. End the session.

    Close PuTTy.
